## Bug 修复：智能问答 500 错误

**问题**：智能问答功能返回 "500 Internal Server Error"，前端显示 "连接失败，请重试"

**根因**：`backend/app/api/chat.py` 中第76行导入并调用了不存在的函数 `verify_token`，实际 `security.py` 中的函数名为 `decode_access_token`。访问 `/api/chat/stream` 时触发 `ImportError` → 500。

**修复内容**：
1. `chat.py`: `verify_token` → `decode_access_token`（核心修复）
2. 移除了死代码 `_build_context` 函数（从未被调用，且内部引用未定义变量 `req.question`）
3. 清理了未使用的导入（`get_current_user`, `User`）
4. 为 `decrypt_api_key` 添加了 try-catch，防止解密异常导致 500

**需要操作**：重启后端服务使修复生效。

## 功能增强：思考过程流式输出

**需求**：LLM 的思考过程（reasoning_content，如 DeepSeek-R1）也要流式输出。

**后端改动** (`backend/app/api/chat.py`):
- 使用 SSE 命名事件区分两种流：
  - `event: reasoning` — 思考过程（`delta.reasoning_content`）
  - `event: message` — 最终回答（`delta.content`）
- 思考→回答切换时发送 `reasoning done` 标记
- 完成事件包含 `full_reasoning` 字段供前端持久化

**前端改动** (`frontend/src/views/ChatView.vue`):
- `Message` 新增 `reasoning` 可选字段
- 使用 `addEventListener('reasoning'/'message')` 监听命名 SSE 事件
- 流式气泡：思考过程用橙色主题 + 可折叠面板，带闪烁光标
- 历史消息：思考过程用灰色可折叠面板，默认展开
