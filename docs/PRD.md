# 产品需求文档（PRD）—— 图像目标检测与分析平台

---

## 一、项目信息

| 项 | 说明 |
|---|---|
| **项目名称** | `yolo_detection_platform` |
| **语言** | 中文 |
| **前端技术栈** | Vue 3 + Vite + TypeScript + Pinia |
| **后端技术栈** | Python FastAPI |
| **数据库** | MySQL（持久化存储）+ Redis（缓存/实时数据） |
| **AI 引擎** | ultralytics YOLO（v8/v10/v11）+ 多模态大模型兼容接口 |
| **部署方式** | 前后端分离，本地/服务器均可部署 |

### 原始需求复述

创建一个图像目标检测网站，支持 YOLO 和多模态大模型协同分析。用户可以上传自定义 YOLO 模型文件、配置自定义大模型 API 接口，在三种工作模式（纯 YOLO / 纯大模型 / YOLO+大模型协同）之间切换，输入源支持图片、视频和摄像头实时采集，分析结果以自然语言描述呈现。

## 二、产品定义

### 2.1 产品目标

| # | 目标 | 衡量标准 |
|---|---|---|
| G1 | 提供灵活的多源输入与多模式 AI 分析管道 | 用户能在 3 步以内完成「输入 → 选择模式 → 获得结果」的核心流程 |
| G2 | 支持用户自定义 AI 模型与接口，降低工具锁定 | 用户可成功上传 YOLO 模型并配置第三方 LLM API 完成协同分析 |
| G3 | 构建直观的结果展示与历史追溯能力 | 检测结果（边界框 + 自然语言描述）清晰可读，历史记录可搜索回溯 |

### 2.2 用户故事

| # | 用户故事 | 验收标准 |
|---|---|---|
| US1 | 作为**研究人员**，我希望上传自定义 YOLO 模型文件，以便检测我研究领域的特定目标 | 上传 `.pt` 模型文件后，系统加载模型并在检测中使用 |
| US2 | 作为**开发者**，我希望配置自己的大模型 API（如 OpenAI 兼容接口），以便获得定制化的自然语言分析 | 填写 API endpoint + key 后，可成功调用并返回分析结果 |
| US3 | 作为**普通用户**，我希望使用「YOLO + 大模型协同」模式，一次性获得目标位置和语义理解 | YOLO 先标出目标区域，大模型针对区域给出自然语言描述 |
| US4 | 作为**学生**，我希望使用摄像头实时检测，以便快速验证模型效果 | 开启摄像头后，画面实时渲染检测框，延迟 < 500ms |
| US5 | 作为**用户**，我希望查看历史检测记录，以便回顾和对比不同配置下的结果 | 历史列表支持按时间/模式筛选，点击可查看完整结果 |

## 三、技术规范

### 3.1 需求池

#### P0 —— 必须实现（MVP）

| 编号 | 需求 | 说明 |
|---|---|---|
| P0-1 | 图片上传 + YOLO 检测 | 上传单张图片，使用内置/自定义 YOLO 模型进行目标检测，渲染边界框 |
| P0-2 | 纯 YOLO 模式 | 仅使用 YOLO 模型检测目标，输出类别名称和置信度 |
| P0-3 | 纯大模型模式 | 仅使用多模态大模型分析图像，返回自然语言描述 |
| P0-4 | YOLO + 大模型协同模式 | YOLO 检测目标区域 → 裁剪区域图像 → 发送给大模型逐区域分析 → 汇总结果 |
| P0-5 | LLM API 配置 | 用户填写 API 地址、密钥、模型名称，支持 OpenAI/Claude 等兼容接口 |
| P0-6 | 检测结果可视化 | 图片上叠加边界框 + 标签，侧栏/下方展示大模型分析文字 |
| P0-7 | MySQL 存储检测记录 | 每次检测保存：时间、输入源、模式、模型配置、结果摘要 |

#### P1 —— 应该实现（迭代 2）

| 编号 | 需求 | 说明 |
|---|---|---|
| P1-1 | 视频上传检测 | 上传视频文件，逐帧/跳帧检测，生成标注视频或帧序列 |
| P1-2 | 摄像头实时采集 | 调用浏览器摄像头，实时推流到后端检测，前端渲染结果 |
| P1-3 | 自定义 YOLO 模型上传 | 用户上传 `.pt` 文件，系统验证格式后注册为可选模型 |
| P1-4 | 历史记录查看 | 列表页查看过往检测记录，支持筛选和详情回看 |
| P1-5 | Redis 缓存 | 缓存模型加载状态、实时检测任务队列、用户 session |

#### P2 —— 可以实现（远期）

| 编号 | 需求 | 说明 |
|---|---|---|
| P2-1 | 批量图片处理 | 一次上传多张图片，批量检测并汇总结果 |
| P2-2 | 结果导出 | 导出标注图片、JSON 检测数据、分析报告 |
| P2-3 | 模型性能对比 | 同一图片对不同模型进行检测，对比耗时和精度 |
| P2-4 | 多 YOLO 模型管理 | 管理已上传的多个模型，支持启用/禁用/删除 |

```
┌──────────────────────────────────────────────────────────┐
│  Logo              图像目标检测平台           [⚙ 设置]    │
├──────────┬───────────────────────────────┬───────────────┤
│          │                               │  检测结果      │
│ 📁 上传   │                               │               │
│ 📷 摄像头 │                               │ ┌───────────┐ │
│ 🎬 视频   │        [ 图像预览区 ]          │ │ 边界框列表  │ │
│          │                               │ │ ✓ car 95%  │ │
│ ──────── │                               │ │ ✓ person   │ │
│ 工作模式  │                               │ └───────────┘ │
│ ○ 纯YOLO │                               │               │
│ ○ 纯LLM  │                               │ 🤖 AI 分析     │
│ ● 协同   │                               │ "图中包含..."  │
│          │                               │               │
│ ──────── │                               │               │
│ 模型选择  │                               │               │
│ YOLO:   │                               │               │
│ [默认v8 ▼]│                              │               │
│ LLM:    │                               │               │
│ [GPT-4o ▼]│                             │               │
│          │                               │               │
│ [ 开始检测 ]                              │               │
└──────────┴───────────────────────────────┴───────────────┘
```

**布局说明**：
- **左侧栏（240px）**：输入源切换（图片/摄像头/视频）、工作模式选择（三选一）、模型选择下拉框、开始检测按钮
- **中央区域（flex: 1）**：图像/视频预览画布，支持缩放和拖拽，边界框直接在画布上渲染
- **右侧面板（360px）**：检测结果分区展示 —— 上半部分为边界框列表（类别 + 置信度），下半部分为大模型分析文字（协同/纯LLM模式下显示）

### 3.3 数据流概要

```
用户输入 → 前端采集 → FastAPI 接收
                         ↓
            ┌─ YOLO-only: 加载模型 → 推理 → 返回 bbox
            ├─ LLM-only:  编码图片 → 调 API → 返回文本
            └─ 协同模式:  YOLO 检测 → 裁剪区域 → LLM 逐区域分析
                         ↓
            MySQL 存储记录 ← Redis 缓存临时结果
                         ↓
            前端渲染 bbox + 展示分析文本
```

### 3.4 待确认问题

| # | 问题 | 影响范围 |
|---|---|---|
| Q1 | 视频检测是否需要实时帧率（如 25fps），还是可以跳帧处理（如每 5 帧检测一次）？ | 后端架构、GPU 资源规划 |
| Q2 | 是否需要用户账号系统（注册/登录）？还是单机工具型，所有人共享？ | 数据库 schema、前端路由 |
| Q3 | 自定义 YOLO 模型文件大小上限？是否限制模型架构版本？ | 上传校验逻辑 |
| Q4 | 是否需要 GPU 推理支持？如果用户无 GPU，是否提供 CPU fallback？ | 部署文档、性能预期 |
| Q5 | 大模型分析的输出期望是纯文本，还是需要结构化 JSON（如按区域编号组织的分析）？ | 后端返回格式、前端渲染 |
| Q6 | 摄像头实时流的传输方式：WebRTC 还是 WebSocket + 帧推送？ | 前后端通信方案 |
